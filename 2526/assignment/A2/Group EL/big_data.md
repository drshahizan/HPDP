### Performance Measurement Function
```python
import os
import time
import psutil

def measure_performance(func, description=""):

    # 1. Preparation
    process = psutil.Process(os.getpid())

    # 2. Baseline snapshots
    ram_start = process.memory_info().rss / (1024 * 1024)
    start_time = time.time()

    # 3. Execute strategy
    result = func()

    # 4. End snapshots
    end_time = time.time()
    ram_end = process.memory_info().rss / (1024 * 1024)

    # 5. Metrics calculation
    duration = round(end_time - start_time, 2)
    memory_used = round(ram_end - ram_start, 2)

    row_count = len(result) if hasattr(result, "__len__") else 0
    throughput = round(row_count / duration, 2) if duration > 0 else 0

    return {
        "Strategy": description,
        "Memory (MB)": memory_used,
        "Time (sec)": duration,
        "Rows Processed": row_count,
        "Throughput (rows/sec)": throughput
    }, result
```

### Explanation

The `measure_performance()` function was created to benchmark each big data handling strategy using a consistent set of performance metrics. It records memory usage before and after execution using the `psutil` library, measures execution time using the `time` module, and calculates throughput based on the number of rows processed per second.

The function returns a dictionary containing memory consumption, execution time, rows processed, and throughput, together with the function result. This enables objective comparison of different big data handling strategies and processing libraries throughout the project.

### 3.2 Chunking

#### Code:

```python
# Chunking reads files in segments so the whole file never hits RAM
def chunking():

    df = pd.read_json(
        "part-01.json",
        lines=True,
        chunksize=100000
    )

    chunk_results = []

    for i, chunk in enumerate(df):

        # Calculate average review length
        avg_text_len = chunk["review_detail"].str.len().mean()
        chunk_results.append(avg_text_len)

        if (i + 1) % 5 == 0:
            print(f"Processed {i + 1} chunks...")

    return pd.Series(chunk_results)


performance2, df_chunk = measure_performance(
    chunking,
    "Chunking"
)

print(performance2)
```

#### Explanation:

Chunking is a technique used to process large datasets that may exceed available system memory. Instead of loading the entire 1.07 GB IMDb dataset at once, the dataset is read in smaller chunks of 100,000 rows using the `chunksize` parameter.

For each chunk, the average character length of the `review_detail` column is calculated. This demonstrates how large-scale text processing tasks can be performed incrementally without exhausting system memory. Progress messages are displayed every five chunks to monitor execution and verify that the dataset is being processed continuously.

### 3.3 Data Type Optimization

#### Code:

```python
def data_type_optimization():

    df = pd.read_json(
        "part-01.json",
        lines=True
    )

    mem_before = (
        df.memory_usage(deep=True).sum()
        / (1024 ** 2)
    )

    # Optimize data types
    df["rating"] = pd.to_numeric(
        df["rating"],
        errors="coerce"
    ).astype("float32")

    df["spoiler_tag"] = (
        df["spoiler_tag"]
        .astype("int8")
    )

    df["movie"] = (
        df["movie"]
        .astype("category")
    )

    df["review_date"] = pd.to_datetime(
        df["review_date"],
        errors="coerce"
    )

    mem_after = (
        df.memory_usage(deep=True).sum()
        / (1024 ** 2)
    )

    print(
        f"Memory before: {mem_before:.1f} MB"
    )
    print(
        f"Memory after : {mem_after:.1f} MB"
    )
    print(
        f"Memory reduction: "
        f"{((mem_before - mem_after) / mem_before) * 100:.1f}%"
    )

    return df


performance3, df_type = measure_performance(
    data_type_optimization,
    "Data Type Optimisation"
)

print(df_type.info())
print(performance3)
```

#### Explanation:

Data Type Optimization reduces memory consumption by converting columns into more efficient data types based on their actual value ranges. The `rating` column is converted from `float64` to `float32`, while the binary `spoiler_tag` column is converted from `int64` to `int8`. The `movie` column is converted to the `category` type to reduce storage requirements for repeated values, and the `review_date` column is converted from string format to `datetime`.

These optimizations improve memory efficiency while preserving the information required for analysis.

### 3.5 Parallel Processing with Polars

#### Code:

```python
def parallel_with_polars():

    df = pl.read_ndjson(
        "part-01.json"
    )

    result = (
        df.select(
            ["rating", "spoiler_tag"]
        )
        .with_columns(
            pl.col("rating")
            .cast(pl.Float32)
        )
        .group_by("spoiler_tag")
        .agg(
            pl.col("rating")
            .mean()
            .alias("avg_rating")
        )
    )

    return result


performance5, polars_parallel_result = (
    measure_performance(
        parallel_with_polars,
        "Parallel Processing with Polars"
    )
)

print(performance5)
```

#### Explanation:

This strategy uses Polars to perform parallel data processing on the large JSONL dataset. The dataset is loaded using `pl.read_ndjson()`, and only the relevant columns (`rating` and `spoiler_tag`) are selected to reduce unnecessary processing. The `rating` column is converted to `Float32` to improve memory efficiency before aggregation.

Polars utilizes parallel execution and query optimization to process data efficiently. The dataset is grouped by `spoiler_tag`, and the average rating is calculated for each group. Compared to traditional single-threaded processing approaches, Polars provides faster execution and better scalability when handling large datasets.
