## 📖 Introduction

In the modern era of aviation and transportation analytics, large-scale flight records are routinely generated by airlines, airports, and regulatory bodies. Efficiently handling this kind of data requires specialised tools and techniques that go beyond what traditional processing libraries can offer.

Our project focuses on mastering scalable big data techniques using tools such as Pandas, Dask, and Polars. We applied various optimisation strategies to load and process a large real-world flight delay dataset, then conducted performance benchmarking across memory usage, execution time, CPU utilisation, and processing throughput.

---

## ✈️ Dataset Overview

- **Name**: Flight Delay Dataset — 2024
- **Source**: [Kaggle – hrishitpatil](https://www.kaggle.com/datasets/hrishitpatil/flight-data-2024)
- **Domain**: Aviation / Transportation
- **File Size**: > 700 MB
- **Shape**: ~7,000,000+ rows × 35 columns

### 📘 Description

This dataset contains comprehensive records of US domestic flight operations throughout 2024, sourced from the Bureau of Transportation Statistics (BTS). It captures detailed information on scheduled vs. actual departure/arrival times, delay causes, airline carriers, and route information — making it ideal for both **delay prediction** and **airline performance analysis**.

### 🔍 Key Features

- Flight identifiers: carrier code, flight number, tail number
- Route info: origin and destination airport, city, state
- Scheduling: scheduled and actual departure/arrival times
- Delay metrics: arrival delay, departure delay (in minutes)
- Delay causes: carrier delay, weather delay, NAS delay, security delay, late aircraft delay
- Flight status: cancelled, diverted flags

### ⚠️ Notes

- Some records may have missing delay cause breakdowns where the flight was not delayed.
- Cancelled and diverted flights may have null arrival/departure time fields.
- Dataset is best suited for time-series and categorical analysis by airline or route.

## ⚙️ Techniques Used

- Selective data loading (column filtering with `usecols`)
- Data type downcasting (reducing int64/float64 footprint)
- Chunk-based reading (processing in batches)
- Sampling methods (random sampling for fast prototyping)
- Parallel processing with Dask
- High-performance computing with Polars

## 📊 Library Benchmarking

Each was tested and compared based on:
