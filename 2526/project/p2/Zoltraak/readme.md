## 📊 YouTube Comment Sentiment Analysis: Cost of Living in Malaysia

### 📖 Project Overview
This project applies a high-performance data processing approach to capture and understand public sentiment regarding the cost of living, inflation, and lifestyle in Malaysia. By extracting and analyzing spontaneous audience feedback from **six selected YouTube videos**, the system classifies public opinion into positive, negative, and neutral sentiments and streams the results through a distributed pipeline built on Apache Kafka and Apache Spark Structured Streaming.

---

### 🎯 Data Source

Comments were collected from six Malaysian-relevant YouTube videos using the **YouTube Data API v3**:

| # | Title | Video ID |
|---|---|---|
| 1 | Cost of living in Malaysia per month - Will you be moving to Malaysia? | `gxCfYO_3trk` |
| 2 | Cost of Living in Malaysia 2026: The Real Numbers | `mFv-7NNYV0w` |
| 3 | Singapore vs Malaysia: Cost of Living 2026 | `8TmWwlrvEDg` |
| 4 | INFLASI \| Singapura Gusar, Malaysia Henti Eksport Ayam | `4RslhMzDmKs` |
| 5 | Pros and Cons of Living in Kuala Lumpur | `DJJvt9_6dNE` |
| 6 | Pros and Cons of living in Malaysia (2025) | `XR6OO2ScEXU` |

| Metric | Value |
|---|---|
| Raw comments scraped | 5,388 |
| After cleaning | 4,472 |
| Languages | English + Bahasa Malaysia |
| Labelling method | VADER sentiment scoring (threshold ±0.05) |

**Label distribution:**

| Sentiment | Count | Percentage |
|---|---|---|
| Positive | 2,115 | 47.35% |
| Neutral | 1,797 | 40.20% |
| Negative | 558 | 12.50% |

---

### 🏗️ System Architecture

The end-to-end pipeline follows a **Lambda-style streaming architecture** across five layers:

```
[ 1. Data Source ]      YouTube Data API v3 → CSV (4,472 comments)
        ↓
[ 2. Ingestion ]        Apache Kafka 3.6.1
                        Topic: youtube-sentiment (3 partitions, port 9092)
        ↓
[ 3. Processing ]       Apache Spark 3.5.2 Structured Streaming (PySpark)
                        JSON parse → Text clean → TF-IDF → SVM UDF → Label + Timestamp
        ↓
[ 4. Storage ]          Elasticsearch 8.19 (Elastic Cloud)
                        Index: youtube_sentiment_index
        ↓
[ 5. Visualization ]    Kibana 8.19 Dashboard
                        Pie chart | Bar chart | Heatmap | Metric panel
```

### 🧠 Machine Learning Models

Both models were trained on a **70 / 20 / 10 stratified split** (3,129 train / 898 test / 443 val):

| Model | Accuracy | Precision | Recall | F1 Score |
|---|---|---|---|---|
| **SVM** (TF-IDF, linear kernel, 5,000 features) | **0.8218** | **0.8197** | **0.8218** | **0.8134** |
| LSTM (64-dim embedding, 10 epochs) | 0.7951 | 0.7896 | 0.7951 | 0.7903 |

**✅ SVM was selected** for pipeline deployment — superior accuracy, lower inference overhead, and no overfitting on this dataset size.

**Per-class breakdown — SVM:**

| Class | Precision | Recall | F1-Score | Support |
|---|---|---|---|---|
| Positive | 0.83 | 0.88 | 0.85 | 425 |
| Neutral | 0.82 | 0.88 | 0.85 | 361 |
| Negative | 0.79 | 0.43 | 0.55 | 112 |

> The Negative class underperforms due to class imbalance (only 12.5% of samples).

---

### ⚡ Batch vs. Streaming Performance

Benchmarked on the same SVM model and 4,472-record dataset:

| Metric | Batch Mode | Streaming Mode |
|---|---|---|
| Total Processing Time (s) | 1.452 | 6.619 |
| Throughput (records/sec) | 3,078.80 | 675.36 |
| Classification Accuracy | 0.923 | 0.908 |
| CPU Usage (%) | 54.5 | 2.0 |
| Memory Usage (MB) | 266.0 | 266.0 |

- **Batch mode** is **4.6× faster** in throughput due to vectorized matrix operations across the full dataset.
- **Streaming mode** provides lower per-record latency and continuous real-time output.
- **Accuracy is consistent** across both modes — no model degradation from the processing paradigm.
- **Memory footprint is identical** — dominated by the static SVM model and TF-IDF vocabulary, not the processing strategy.

### 📈 Key Findings — Kibana Dashboard

The pipeline processed a total of **22,360 records** across all streaming runs:

| Sentiment | Count | Percentage |
|---|---|---|
| Positive | ~10,880 | 48.68% |
| Neutral | ~9,470 | 42.37% |
| Negative | ~2,000 | 8.94% |

**Insights:**
- Nearly half of all comments (48.68%) expressed **positive sentiment**, reflecting generally favorable views toward Malaysia as a place to live and visit.
- The 42.37% **neutral proportion** represents a large body of informational or observational commentary that carries no strong emotional polarity.
- Negative comments at 8.94% (~2,000 records) are a minority but represent actionable feedback for policymakers and tourism stakeholders.
- The heatmap confirms the model performs best on Positive and Neutral classes, with the Negative class being hardest to classify due to class imbalance.

### 🚀 How to Run

> All notebooks run on **Google Colab**. No local installation required.

| Step | Action | Notebook |
|---|---|---|
| 1 | Collect YouTube comments | `youtube_comments_extractor_multi.ipynb` |
| 2 | Clean data & assign VADER labels | `datacleaning_positive_negative_latest.ipynb` |
| 3 | Train SVM + LSTM, compare results | `model_training.ipynb` |
| 4 | Start Kafka, create topic | `kafka_producer_colab.ipynb` (Cells 0–4) |
| 5 | Run Spark streaming → Elasticsearch | `spark_streaming_colab.ipynb` (Cells 0–13) |
| 6 | Run performance benchmark | `batch_vs_streaming.ipynb` |
| 7 | Import dashboard to Kibana | Import `export.ndjson` via Stack Management → Saved Objects |

**Prerequisites:**
- Google account (for Colab)
- YouTube Data API v3 key ([get one here](https://console.cloud.google.com/))
- Elastic Cloud free trial at [cloud.elastic.co](https://cloud.elastic.co) — select **Hosted**, version **8.19**
- Elastic Cloud API key (deployment → Overview → API Keys → New)
