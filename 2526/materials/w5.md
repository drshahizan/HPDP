# Week 5: Five Core Strategies for Big Data Handling**

### **1. Load Less Data**
[cite_start]The primary defense against Out-of-Memory (OOM) errors is to be selective at the point of ingestion[cite: 50].
* [cite_start]**Definition**: Reading only the specific columns or rows required for your analysis[cite: 14, 15].
* [cite_start]**Implementation**: Use the `usecols` parameter to filter columns and `nrows` to limit rows during the initial load[cite: 44, 45].
* [cite_start]**Why it matters**: It prevents irrelevant data from consuming RAM and significantly reduces load times for "wide" datasets[cite: 48, 49].
* **Code Example**:
  ```python
  import pandas as pd
  # Only load columns A, B, and C for the first 500,000 rows
  [cite_start]df = pd.read_csv("big_data.csv", usecols=["col_A", "col_B", "col_C"], nrows=500_000) [cite: 54, 56, 58, 59]
  ```

### **2. Chunking**
[cite_start]Chunking allows for the processing of files that are larger than the available system RAM[cite: 71].
* [cite_start]**Definition**: Breaking a large dataset into small, manageable pieces (chunks)[cite: 19, 20].
* [cite_start]**Implementation**: Use the `chunksize` parameter in Pandas to iterate through a file[cite: 67].
* [cite_start]**Why it matters**: It keeps memory usage constant, regardless of the total file size, by processing one piece at a time and releasing it before loading the next[cite: 68, 72].
* **Code Example**:
  ```python
  for chunk in pd.read_csv("big_data.csv", chunksize=100_000):
      # [cite_start]Process each 100k-row chunk independently [cite: 79, 80]
  ```

### **3. Data Type Optimisation**
[cite_start]Standard data loaders often default to memory-heavy types (like `int64` or `float64`) even when smaller types would suffice[cite: 89, 90].
* [cite_start]**Definition**: Downcasting numeric types (e.g., `int64` → `int32`) and converting repetitive strings to the `category` type[cite: 89, 91].
* [cite_start]**Why it matters**: This can reduce the dataset's memory footprint by 50% or more and speeds up downstream operations[cite: 93, 94].
* **Code Example**:
  ```python
  df["age"] = pd.to_numeric(df["age"], downcast="integer")
  [cite_start]df["cat"] = df["cat"].astype("category") [cite: 101, 103]
  ```

### **4. Sampling**
[cite_start]Sampling is used to accelerate the development cycle without sacrificing statistical integrity[cite: 115].
* [cite_start]**Definition**: Working on a smart, representative subset of the data[cite: 30, 31].
* [cite_start]**Techniques**: Includes random sampling and **stratified sampling**, which preserves the ratios of specific categories[cite: 111, 112].
* [cite_start]**Why it matters**: It makes Exploratory Data Analysis (EDA) feasible on massive datasets and allows for rapid code testing before scaling up[cite: 116, 119].

### **5. Parallel Processing with Scalable Libraries**
[cite_start]When Pandas becomes a bottleneck, parallel processing unlocks the full power of multi-core hardware[cite: 137, 141].
* [cite_start]**Definition**: Utilizing libraries designed for distributed or out-of-core computing[cite: 36, 131].
* **Key Libraries**:
    * [cite_start]**Dask**: Offers a Pandas-like API that distributes tasks across all available CPU cores[cite: 133, 145].
    * [cite_start]**Polars**: A blazing-fast library written in Rust that uses "lazy evaluation" to optimize queries[cite: 134, 148].
    * [cite_start]**PyArrow**: Uses a columnar format for highly efficient I/O and analytics[cite: 135].

## **Key Takeaways**
* [cite_start]**Be Selective**: Always load only what you need; unused columns are immediate memory waste[cite: 186, 187].
* [cite_start]**Be Efficient**: Right-sizing data types is one of the easiest ways to prevent crashes[cite: 192, 193].
* [cite_start]**Scale Smartly**: Use sampling for development and parallel libraries like **Polars** or **Dask** for production-grade speed[cite: 196, 198, 199].
