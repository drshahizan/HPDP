# Week 5: Five Core Strategies for Big Data Handling 

In modern data environments, datasets often exceed hundreds of megabytes or even gigabytes, requiring not only efficient programming techniques but also strategic thinking in data processing design.

## **1. Load Less Data (Selective Data Ingestion)**

This strategy emphasizes minimizing data at the earliest stage of processing.

In real-world scenarios, datasets often contain many irrelevant features. Loading all columns without filtering leads to unnecessary memory consumption and slower computation.

Advanced Practice:

* Use `usecols` for column selection
* Use `nrows` for quick prototyping
* Apply filtering conditions during loading (e.g., SQL queries or PyArrow dataset filters)

Example:

```python
df = pd.read_csv("big_data.csv", usecols=["col_A", "col_B"], nrows=500000)
```

Extended Insight:
Selective loading is especially important in **data pipelines**, where early reduction reduces downstream computational cost. This aligns with the principle of **data minimization in big data architecture**.


## **2. Chunking (Out-of-Core Processing)**

Chunking enables processing datasets that do not fit into memory.

Instead of loading the entire dataset, data is processed in smaller partitions sequentially.

Example:

```python
for chunk in pd.read_csv("big_data.csv", chunksize=100000):
    process(chunk)
```

Advanced Practice:

* Combine chunking with aggregation (e.g., incremental sum, mean)
* Store intermediate results instead of keeping everything in memory
* Use generators for memory-efficient workflows

Example (Aggregation):

```python
total = 0
for chunk in pd.read_csv("big_data.csv", chunksize=100000):
    total += chunk["sales"].sum()
```

Extended Insight:
Chunking is a core concept in **stream processing systems** and is widely used in platforms such as Apache Spark and Hadoop.


## **3. Data Type Optimisation (Memory Engineering)**

Efficient data representation significantly impacts performance.

Default data types are often over-allocated, leading to excessive memory usage.

Example:

```python
df["age"] = pd.to_numeric(df["age"], downcast="integer")
df["category"] = df["category"].astype("category")
```

Advanced Practice:

* Convert `object` to `category` when values repeat
* Use `float32` instead of `float64` where precision allows
* Monitor memory using:

```python
df.info(memory_usage="deep")
```

Extended Insight:
Data type optimisation is critical in **columnar storage systems** such as Parquet and Arrow, where efficient encoding improves both storage and query performance.


## **4. Sampling (Scalable Experimentation)**

Sampling reduces dataset size while preserving statistical characteristics.

This is especially useful during:

* Exploratory Data Analysis (EDA)
* Model prototyping
* Debugging pipelines

Example:

```python
df_sample = df.sample(frac=0.1, random_state=42)
```

Advanced Practice:

* Use **stratified sampling** for imbalanced datasets
* Compare results between sample and full dataset
* Gradually scale from sample → full data

Extended Insight:
Sampling supports **iterative development workflows**, allowing faster experimentation cycles without sacrificing analytical validity.


## **5. Parallel Processing with Scalable Libraries**

When data size or computation grows, single-threaded Pandas becomes inefficient.

Modern libraries support parallel and distributed computation.

### Key Libraries:

**Dask**

```python
import dask.dataframe as dd
df = dd.read_csv("big_data.csv")
df.groupby("col").mean().compute()
```

**Polars**

```python
import polars as pl
df = pl.read_csv("big_data.csv")
df.select(pl.col("col").mean())
```

**PyArrow**
Used for fast I/O and columnar memory format.



### Advanced Concepts:

Lazy Evaluation:

* Polars optimizes execution plans before running queries

Task Scheduling:

* Dask builds computation graphs and executes in parallel

Distributed Computing:

* Enables scaling across multiple machines (cluster computing)



## **Performance Comparison Overview (Enhanced)**

| Strategy            | Memory Impact | Speed Gain | Use Case              | Limitation                     |
| ------------------- | ------------- | ---------- | --------------------- | ------------------------------ |
| Load Less Data      | High          | Medium     | Feature selection     | May miss useful data           |
| Chunking            | Very High     | Medium     | Large file processing | More complex logic             |
| Type Optimisation   | Very High     | Medium     | Memory reduction      | Requires understanding of data |
| Sampling            | Very High     | Very High  | EDA, testing          | Not suitable for final results |
| Parallel Processing | High          | Very High  | Large-scale pipelines | Setup complexity               |


## **Common Pitfalls (Very Important for Students)**

Many students fail not because they do not know the tools, but because they misuse them.

Common mistakes include:

* Loading full dataset without filtering
* Ignoring data types (using default `object`)
* Using Pandas for very large datasets without scaling
* Not validating sample results with full dataset
* Writing inefficient loops instead of vectorized operations

## **Best Practices for HPDP Assignment**

Students should structure their workflow as follows:

Step 1: Load a small subset using sampling
Step 2: Analyze memory usage and optimize data types
Step 3: Apply chunking for full dataset processing
Step 4: Implement solution using Pandas
Step 5: Extend using Dask / Polars / PyArrow
Step 6: Compare performance (time and memory)



## **Key Takeaways**

Efficient Big Data handling is not about using powerful hardware alone, but about applying the right strategies at the right stage.

Students must understand that:

* Memory is the primary constraint in Big Data processing
* Optimisation should start from data ingestion
* Scalability requires both algorithmic and system-level thinking
* Parallel computing is essential for production-level systems
